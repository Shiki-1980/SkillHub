# 对抗训练：增强模型鲁棒性

## 方法

使用 Task C 生成的对抗样本来增强 XGBoost 分类器的训练数据：

1. **80/20 划分**：8,400 训练 / 2,101 测试
2. **生成对抗变体**：对训练集中的 malicious/unsafe skill，每个生成 3 个变体（随机 1-3 个逃逸算子）
3. **训练两个模型**：
   - Vanilla XGBoost：仅原始训练数据
   - Robust XGBoost：原始 + 对抗变体（保持原标签）
4. **评估**：在三个测试场景下对比
   - Clean：原始测试集
   - Mixed：50% 原始 + 50% 对抗变体
   - Adversarial：纯对抗变体

## 实验结果

| 测试场景 | Vanilla XGBoost | Robust XGBoost | Δ |
|---|---|---|---|
| Clean (原始) | **0.871** | 0.566 | −0.305 |
| Mixed (50% 原始+50% 对抗) | 0.228 | **0.643** | **+0.415** |
| Adversarial (纯对抗) | 0.000 | **0.489** | **+0.489** |

## 分析

1. **Vanilla 模型对对抗样本完全无效**：纯对抗样本上 F1=0.000，说明 Task C 的扰动算子成功生成了检测器无法识别的变体
2. **对抗训练显著提升鲁棒性**：Robust 模型在对抗样本上恢复 0.489 F1（+0.489），在混合场景提升 0.415
3. **经典的鲁棒性-精度权衡**：Clean 场景下 Robust 模型 F1 下降（0.871→0.566），这是因为对抗训练引入了训练分布的偏移。该权衡在对抗机器学习文献中广泛记录 [Goodfellow+ 2015]
4. **闭环验证**：Task C（攻击）→ 对抗训练（防御）→ 鲁棒性提升，构成了完整的红蓝对抗闭环
