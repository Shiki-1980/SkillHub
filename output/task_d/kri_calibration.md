# KRI 风险评分权重校准

## 方法

KRI 复合风险评分模型定义为：

**KRI = w_t × Threat + w_i × Impact + w_e × Exposure − SafetyBonus**

原始权重（0.40/0.35/0.25）为经验设定。通过网格搜索在 [0.15, 0.55] 范围内（步长 0.05），对使 KRI 与异常标签（malicious/unsafe vs normal）F1 最大的权重组合进行全局搜索。对每个权重组合，同步搜索最优 KRI 阈值 [0.10, 0.78]（步长 0.02）。

搜索空间：w_t + w_i + w_e = 1.0，共 ~100 个有效组合。

## 分析

1. **Threat 是最强信号**：w_t 从 0.15 到 0.55，F1 持续提升（+5.2%）。最优配置中 w_t=0.55，远超原值 0.40。这是因为危险关键词（eval/exec/curl pipe to shell 等）与异常标签的相关性远强于权限范围和外部暴露。

2. **Impact 与 Exposure 应降低**：最优配置中 w_i=0.25（原 0.35）、w_e=0.20（原 0.25）。权限范围和外部服务依赖在 DataHub 平台上普遍存在，对异常的分辨力较弱。

3. **增益小但稳健**：F1 从 0.239 到 0.248（+3.6% 相对提升），增幅不大。但所有 10 个最优配置的 w_t 都在 0.45-0.55 范围，说明 Threat 优先的规律是稳定的。

4. **稳定性**：F1 范围仅 0.230-0.248（标准差 0.005），说明 KRI 评分对权重变化不敏感——主要瓶颈是特征本身的判别力，而非权重配置。

5. **安全性权衡**：较高的 w_t 意味着 KRI 对危险操作更敏感，这在对安全性要求高的场景中是合理的——宁可高估 Threat 也不应低估。

## 结论

建议更新 KRI 权重为 **w_t=0.55, w_i=0.25, w_e=0.20**（阈值 0.36）。此配置将 KRI 与异常标签的 F1 从 0.239 提升至 0.248。核心启示：**在 skill 安全评估中，操作威胁信号比权限范围和外部暴露更具判别力**。
