# 无监督方法集成：IF + LOF + LSTM Autoencoder 共识投票

## 方法

将三个无监督异常检测方法的预测结果进行集成，评估四种投票策略：

| 策略 | 规则 | 适用场景 |
|---|---|---|
| **Any (≥1/3)** | 任一方法标记即异常 | 最大化召回，宁可误报不漏报 |
| **Majority (≥2/3)** | 多数同意才异常 | 平衡精确率和召回 |
| **Unanimous (3/3)** | 全票通过才异常 | 最高置信度，低误报 |
| **Weighted** | 按各自 F1 加权求和 | 利用历史性能信息 |

## 实验结果

| 方法 | F1 | Precision | Recall | 标记数 |
|---|---|---|---|---|
| Isolation Forest | 0.082 | 0.087 | 0.077 | 1,050 |
| LOF | 0.095 | 0.101 | 0.090 | 1,050 |
| LSTM Autoencoder | 0.046 | 0.051 | 0.043 | 983 |
| **Any (≥1/3)** | **0.118** | 0.086 | 0.185 | 2,527 |
| Majority (≥2/3) | 0.033 | 0.055 | 0.024 | 511 |
| Unanimous (3/3) | 0.002 | 0.022 | 0.001 | 45 |
| Weighted (F1) | 0.094 | 0.087 | 0.102 | 1,377 |

## 方法一致性矩阵

| 交集 | 数量 | 占比 |
|---|---|---|
| IF ∩ LOF | 141 | 2.6% |
| IF ∩ LSTM | 372 | 6.8% |
| LOF ∩ LSTM | 88 | 1.6% |
| 三者共识 | **45** | **0.8%** |

## 分析

1. **"Any" 策略最优**：F1=0.118，比最佳单独方法（LOF, 0.095）提升 24%。通过最大化召回（0.185）换取整体 F1 提升。

2. **三方法高度互补**：三者同时标记的仅 45 条（0.8%），说明它们捕获了异常的不同维度——IF 偏向全局离群、LOF 偏向局部密度异常、LSTM AE 偏向重构误差大的样本。

3. **严格的投票策略效果差**：Unanimous (F1=0.002) 和 Majority (F1=0.033) 远低于单独方法，因为三个方法的一致率太低，严格投票导致大量漏报。

4. **加权策略与单独 LOF 持平**：Weighted F1=0.094 ≈ LOF 0.095，说明简单加权未能有效利用互补信息——因为 F1 作为权重在高一致性场景才有用，而这里恰好是低一致性场景。

5. **集成提升有限**：即使最优的 Any 策略 F1=0.118，仍远低于有监督 XGBoost（0.829）。这再次验证：**弱标签对无监督异常检测的评估存在根本性局限**——弱标签本身无法充分定义"异常"。

## 结论

三个无监督方法确实互补（共识仅 0.8%），简单的 "Any" 投票策略可提升 F1 至 0.118（+24%）。但因弱标签质量有限，集成效果仍未达实用水平。未来若获得高质量标注数据，可重新评估集成的实际价值。
